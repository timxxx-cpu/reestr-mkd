# 19. Подвалы: целевая модель и правила (актуально)

## 1) Базовая модель

Подвал хранится как **отдельный блок** в `building_blocks`:

- `is_basement_block` — признак подвального блока;
- `building_id` — прямая связь с конкретным зданием;
- `linked_block_ids` — к каким обычным блокам здания относится подвал;
- `basement_depth` — глубина;
- `basement_has_parking` и `basement_parking_levels` — параметры парковки в подвале.

`floors.basement_id` ссылается на `building_blocks(id)` basement-блока.

## 2) Очистка reset_schema

В `db/reset_schema.sql` сделан полный сброс через:

- `drop schema if exists public cascade;`
- `create schema public;`

Это гарантирует отсутствие «хвостов» (таблиц, индексов, триггеров, функций, последовательностей).

## 3) Правила по типам объектов

Реализованы и/или зафиксированы следующие правила:

1. **Подземный паркинг**
   - там один парковочный блок, и он сам является подземной частью;
   - отдельные basement-блоки не создаются;
   - этажи `-1..-N` формируются из самого parking-блока по `levels_depth`.

2. **Наземный паркинг из легких конструкций / открытый**
   - basement-блоки запрещены.

3. **Наземный паркинг капитальный**
   - basement-блоки допускаются.

4. **Инфраструктура**
   - всегда один обычный блок;
   - допускается не более одного basement-блока.

## 4) Создание подвалов при создании здания

Логика в `composition-routes`:

- при создании/обновлении здания можно передать `basementsCount` (или `basementCount`);
- basement-блоки создаются сразу вместе со зданием;
- на момент создания явной связи с блоками может не быть (`linked_block_ids=[]`),
  **кроме** случая, когда у здания ровно один обычный блок — тогда связь ставится автоматически.

## 5) Связь для дома из одного блока

Если в здании ровно один обычный блок и есть basement-блок(и), связь `linked_block_ids` устанавливается автоматически в `[<этот_блок>]`.

## 6) Где на backend это применяется

- `composition-routes` — создание/обновление basement-блоков по `basementsCount`;
- `project-extended-routes` — сохранение `buildingDetails[*_features].basements` в basement-блоки и авто-связь для single-block зданий;
- `floor-generator` — генерация подвальных этажей из basement-блоков;
- `GET /api/v1/projects/:projectId/basements` и `PUT /api/v1/basements/:basementId/parking-levels/:level` работают через `building_blocks`.
