# 28. Fullscreen CAD редактор блоков: фронтенд и варианты backend

## Что внедрено во frontend

- Редактирование `blockGeometry` переведено в полноэкранный CAD-режим (fullscreen overlay).
- Подложка карты отключена: редактор работает на чистом CAD полотне.
- Добавлена метровая сетка (мелкая 1м и акцент каждые 10м).
- Контур здания автоматически ориентируется горизонтально по самой узкой стороне.
- Контур блока рисуется/правится в этой же системе координат.
- Сохранение сохраняет GeoJSON обратно в исходной географической системе координат.

## Реализованные инварианты

1. Контур блока не может быть сохранён, если меньше 3 точек.
2. Контур блока проверяется на вхождение внутрь геометрии здания.
3. Узлы контура привязываются к метровой сетке (округление до 1 м).

## Варианты backend реализации

### Вариант A (быстрый, совместимый с текущей моделью)

- Оставить текущий контракт сохранения `block.footprint_geojson`.
- Frontend передаёт целую геометрию блока после редактирования.
- Backend валидирует:
  - корректность GeoJSON,
  - containment в `building.footprint_geojson`,
  - ограничение максимального числа вершин.

**Плюсы**: минимум изменений в API и БД.
**Минусы**: нет истории операций CAD, нет серверных снап-правил.

### Вариант B (доменный CAD endpoint)

Добавить endpoint-ы:

- `POST /api/v1/blocks/{blockId}/cad/preview-validate`
- `PUT /api/v1/blocks/{blockId}/cad/geometry`

Где `preview-validate` возвращает:

- ошибки геометрии,
- предупреждения по качеству контура,
- серверные метрики площади/периметра.

**Плюсы**: единая серверная валидация CAD-правил.
**Минусы**: дополнительная разработка backend-domain.

### Вариант C (event-sourcing CAD)

Ввести таблицы:

- `block_cad_sessions`
- `block_cad_events` (add_vertex / move_vertex / delete_vertex / close_ring / save)

Сборка финальной геометрии производится backend-агрегатором.

**Плюсы**: аудит, replay, откат к любой точке.
**Минусы**: самый дорогой по срокам и сопровождению вариант.

## Рекомендованный путь внедрения

1. Стартовать с варианта **A** (быстрая доставка ценности).
2. После стабилизации перейти к варианту **B**.
3. Вариант **C** вводить только при реальной потребности в полном аудите CAD-операций.
